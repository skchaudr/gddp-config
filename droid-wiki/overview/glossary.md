# Glossary

| Term | Definition |
|---|---|
| **GDDP** | Graph-Driven agentic Development. The system where project graphs define what work to do, in what order, and what counts as done. |
| **Graph truth** | The canonical state of a project graph (node statuses, dependencies, acceptance). Human-owned. The runtime cannot mutate it. |
| **Node** | A capability, milestone, or constraint in a project graph. Defined in `schemas/v1/node.yaml`. Has acceptance criteria, dependencies, and status. |
| **Project graph** | A directory under `graphs/` containing a `project.yaml` index and per-node YAML files. Represents one project managed by GDDP. |
| **Acceptance criteria** | Verifiable conditions that a node's work must satisfy. Each has a stable kebab-case `id` and a `criterion` string. |
| **Event** | Something that happened (GitHub webhook, manual trigger). Normalized into `schemas/v1/event.yaml` before routing. |
| **Job** | Bounded work derived from an event and a node. Defined in `schemas/v1/job.yaml`. Has constraints, acceptance criteria, and a status. |
| **Task packet** | The exact payload sent to an executor. Defined in `schemas/v1/task_packet.yaml`. Archived regardless of outcome. |
| **Result** | What an executor returns. Defined in `schemas/v1/result.yaml`. Contains changed files, acceptance check, risks, and follow-up candidates. |
| **Receipt** | A reviewable record of an executor's work. The runtime produces receipts; humans decide whether to advance graph truth based on them. |
| **Queue record** | A SQLite record tracking job lifecycle (intake, classified, ready, running, awaiting_result, awaiting_review, complete, failed, etc.). |
| **Artifact verification** | A hard gate: all required artifacts in a node must verify before the node advances. One record per artifact, per job. |
| **Shape profile** | Optional context encoding what a project type should look like structurally. Does not mutate graph truth. |
| **Return path** | The flow from executor result to review receipt to human decision. Contrast with forward path (event to job to executor). |
| **Decision loop** | The runtime module that reads events, reasons over the graph, and dispatches jobs. Lives in `gddp-runtime`. |
| **Executor** | A worker that performs a job: jules, vertex, pi_worker, vm_worker, or human. |
| **Deterministic verifier** | `scripts/verify_node.py`. Checks what can be mechanically proven (symbol presence, file existence, pattern matching). No LLM, no network. |
| **Semantic evaluation** | A later judgment layer that goes beyond deterministic checks. Uses LLM reasoning to evaluate intent preservation and graph integrity. |
| **Pi evaluator harness** | A setup that drives the semantic evaluator through the pi coding agent for live visibility. |
| **Graphify** | A tool that extracts a graph skeleton from code. Output is used by `scripts/graphify_to_nodes.py` for brownfield project bootstrapping. |
| **Natural bounded autonomy** | An agent guard system (`.agents/`) that uses natural language as the control plane, with git-checkpoint safety nets. |
| **Handoff** | A markdown file in `.handoffs/` that captures session state for the next agent. Numbered sequentially from a template. |
| **Shareable graph bundle** | A single YAML file containing project metadata and all node documents inline. Generated by `scripts/export_graph_bundles.py`. |
